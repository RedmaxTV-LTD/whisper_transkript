# 06 — Подготовка к коммиту

Используй этот промт перед commit в проекте Whisper STT sidecar.

---

# Промт

Подготовь проект Whisper STT sidecar к commit.

Проведи проверку изменений, cleanup и validation перед фиксацией в git.

---

# Контекст проекта

```text
whisper-stt (api) → Redis → whisper-worker (GPU)
```

Критическое правило:

```text
HTTP API must never block on GPU transcription.
```

---

# Главная задача

Перед commit:

* убрать мусор и debug-код;
* проверить imports и типы;
* проверить Docker / env consistency;
* убедиться, что секреты не попадут в commit;
* проверить docs при изменении поведения;
* не коммитить без явной просьбы пользователя — только подготовить и предложить message.

---

# Проверь backend (`app/`)

* нет GPU/STT в `main.py`;
* worker корректно обрабатывает ошибки jobs;
* нет unused imports / dead code;
* нет debug prints;
* Pydantic models согласованы;
* новые env добавлены в `settings.py` + `.env.example` + compose.

---

# Проверь Redis / jobs

* dedup lock release в finally;
* статусы согласованы с `job_models.py`;
* queue self-heal не сломан;
* recovery logic intact.

---

# Проверь Docker

* `docker-compose.yml` валиден;
* api и worker с одинаковыми критичными env;
* volumes на месте;
* не закоммичены `.env`, `google_key.json`, модели.

---

# Проверь security

* нет токенов в коде;
* `.gitignore` покрывает `.env`, logs, models, keys;
* логи не содержат Bearer tokens.

---

# Проверь documentation

Актуальность при изменениях:

* `README.md`
* `docs/architecture/*`
* `docs/changelog/CHANGELOG.md`

---

# Git readiness

* нет временных файлов (`Untitled`, `*.pyc`, `__pycache__`);
* нет accidental large binaries (model.bin);
* осмысленный commit message (1–2 предложения, focus on why).

---

# Verification commands

```bash
cd /opt/whisper
docker compose config
docker compose up --build -d
curl -s http://localhost:19900/health | jq .
docker compose logs whisper-worker --tail=20
# опционально: тестовый POST /transcribe
```

# Важное правило

Критические проблемы (GPU в API, broken queue) — не маскировать; описать и исправить до commit.
