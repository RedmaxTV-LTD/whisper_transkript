# Контекст проекта

Проект: **Whisper STT sidecar** (v1.0.0).

GPU-сервис транскрипции записей колл-центра: клиент ставит задачу через API, worker обрабатывает на GPU, результат — через polling jobs.

Архитектура:

```text
Client (n8n, CRM)
  ↓ POST /transcribe
whisper-stt (FastAPI, role=api)
  ↓ Redis LPUSH
whisper-worker (role=worker, GPU)
  ↓ faster-whisper [+ pyannote]
  ↓ result в Redis
Client ← GET /jobs/by-dedup/{dedup_key}
```

Стек: Python, FastAPI, Redis, faster-whisper, pyannote (опц.), Docker Compose, CUDA.

---

# Критическое правило проекта

```text
HTTP API must never block on GPU transcription.
```

Новая фича не должна:

* выполнять распознавание в API-контейнере;
* блокировать `POST /transcribe` на время GPU-работы;
* ломать dedup и очередь Redis;
* нарушать разделение api/worker без явного обоснования.

---

# Перед реализацией

Проанализируй:

* относится ли фича к API, worker или обоим;
* нужны ли изменения в `TranscribeBody` / `TranscribeResponse`;
* влияет ли на `dedup_key` или статусы jobs;
* нужны ли новые env-переменные в `settings.py` и `docker-compose.yml`;
* нужна ли hot-reload конфигурация (как spelling/speaker_roles).

Если фича требует синхронного GPU в HTTP — предложи асинхронную альтернативу через очередь.

# Правила реализации

* Следуй существующим паттернам в `app/`.
* GPU и ffmpeg — только в worker (`ThreadPoolExecutor`).
* Новые шаги пайплайна — новый `status` / `current_step` + обновление `job_models.py`.
* Валидация запроса — Pydantic в `transcribe_body.py`.
* Не ломай обратную совместимость API без явного указания.
* Минимальный scope — только то, что нужно для фичи.

# Проверка

После реализации проверь:

* `POST /transcribe` возвращает job без блокировки;
* worker обрабатывает задачу до `completed`;
* `GET /jobs/by-dedup/{dedup_key}` отдаёт result;
* dedup не создаёт дубликаты для тех же URL;
* OpenAPI `/docs` отражает новые поля.

Команды:

```bash
curl -s http://localhost:19900/health
# POST /transcribe с тестовым телом
# GET /jobs/{job_id}
docker compose logs whisper-worker --tail=50
```
