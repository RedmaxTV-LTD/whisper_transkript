# 03 — Рефакторинг

Используй этот промт для безопасного рефакторинга Whisper STT без изменения внешнего поведения API и семантики транскрипции.


---

# Контекст проекта

Проект: **Whisper STT sidecar**.

```text
whisper-stt (api) → Redis → whisper-worker (GPU) → result
```

Модули: `main.py`, `worker.py`, `transcribe_pipeline.py`, `job_store.py`, `transcribe_engine.py`, …

---

# Критическое правило

```text
HTTP API must never block on GPU transcription.
```

```text
НЕ МЕНЯЙ БИЗНЕС-ЛОГИКУ без явного указания.
```

Рефакторинг не должен менять:

* формат `POST /transcribe` и ответа jobs (если не задано иначе);
* алгоритм `dedup_key`;
* семантику статусов jobs;
* маппинг RX/TX → client/operator;
* правила sync fallback.

Если поведение неизбежно меняется — явно опиши это.

# Разрешено

* разделить большие файлы;
* вынести повторяющийся код;
* улучшить типизацию и naming;
* упростить `job_store` / pipeline без смены контрактов;
* улучшить logging и error messages;
* оптимизировать Redis-операции (с сохранением семантики);
* уменьшить связанность модулей.

---

# Запрещено

* переносить GPU-работу в API;
* менять dedup / queue protocol без миграции;
* удалять recovery / watchdog / self-heal;
* ломать backward compatibility API;
* добавлять MySQL, RQ, frontend «заодно»;
* over-engineering (лишние абстракции, microservices).

---

# Области рефакторинга

## API layer (`main.py`)

Routers только: auth, validation, Redis, response mapping.

## Worker (`worker.py`)

BRPOP loop, semaphores, heartbeat, job lifecycle.

## Pipeline (`transcribe_pipeline.py`)

Оркестрация шагов; GPU — через executor.

## Redis (`job_store.py`)

Ключи, locks, queue heal — без дублирования в main/worker.

## Engine (`transcribe_engine.py`, `diarize_engine.py`)

STT и diarization; без HTTP-зависимостей.
