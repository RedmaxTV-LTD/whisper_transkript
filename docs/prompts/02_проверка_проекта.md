# 02 — Проверка проекта

Используй этот промт для комплексной проверки проекта Whisper STT после реализации фич, рефакторинга или изменений инфраструктуры.

---

# Промт

Проведи полную техническую проверку проекта Whisper STT sidecar.

---

# Контекст проекта

Проект: **Whisper STT sidecar**.

GPU-sidecar для транскрипции записей колл-центра через асинхронную очередь Redis.

Архитектура:

```text
Client
  ↓ POST /transcribe
whisper-stt (FastAPI, role=api)
  ↓ Redis
whisper-worker (GPU, faster-whisper)
  ↓ GET /jobs/*
```

Docker Compose: `redis`, `whisper-stt`, `whisper-worker`.

---

# Критическое правило проекта

```text
HTTP API must never block on GPU transcription.
```

Проверяй, что изменения не:

* загружают Whisper в API-контейнер;
* выполняют STT/ffmpeg в route handlers;
* ломают очередь и dedup;
* создают race conditions между api и worker.

---

# Основная задача проверки

Найти:

* архитектурные нарушения (GPU в API, sync transcribe);
* проблемы Redis (очередь, dedup lock, orphan jobs);
* проблемы worker (семафор, heartbeat, stale, recovery);
* ошибки пайплайна (dual-track, sync, diarization);
* проблемы Docker / env / GPU;
* проблемы безопасности (секреты, токены);
* устаревшую документацию.

---

# Проверь Backend (`app/`)

* `main.py` — только постановка задач и статус, без GPU;
* `worker.py` — BRPOP, семафор, heartbeat, recovery;
* `transcribe_pipeline.py` — все режимы (url, rx/tx, sync);
* `job_store.py` — атомарность, self-heal очереди;
* `transcribe_engine.py` — модель, dual-track, spelling;
* `diarize_engine.py` — загрузка до BRPOP;
* `settings.py` — все env прочитаны корректно.

---

# Проверь архитектурные нарушения

Нельзя:

* faster-whisper / pyannote в `main.py`;
* блокирующий ffmpeg в async route без executor;
* дублирование логики pipeline в API и worker;
* изменение `dedup_key` без обновления docs;
* игнорирование terminal/active статусов в dedup.

---

# Проверь Redis и jobs

* `whisper:queue` синхрон с `queued` jobs;
* recovery при рестарте worker (`WHISPER_WORKER_QUEUE_FLUSH_ON_START`);
* `worker_orphaned` для прерванных задач;
* stale watchdog (`WHISPER_JOB_STALE_SEC`);
* `whisper:worker:alive` и `worker_available` в API;
* dedup lock и повторный POST.

---

# Проверь worker и GPU

* `WHISPER_MAX_CONCURRENT_JOBS` и семафор;
* heartbeat при ожидании слота (`waiting_gpu`);
* pyannote load order при `WHISPER_DIARIZATION=1`;
* cleanup temp WAV после job;
* обработка ошибок → `failed` с понятным `error`;
* ThreadPoolExecutor sizing.

---

# Проверь пайплайн транскрипции

* single `url`: mono/stereo, diarize on/off;
* dual rx/tx: `call_direction`, роли RX/TX;
* sync auto/manual/off и fallback;
* spelling fixes hot-reload;
* speaker roles catalog;
* лимит скачивания и timeout.

---

# Проверь API

* валидация `TranscribeBody`;
* 202 vs 200 при dedup;
* `GET /jobs/by-dedup/{dedup_key}` (URL-encoded);
* `client_hint`, `retry_recommended`;
* Bearer auth при `WHISPER_API_TOKEN`;
* OpenAPI examples актуальны.

---

# Проверь Docker

* `docker-compose.yml`: redis, whisper-stt, whisper-worker;
* `WHISPER_PROCESS_ROLE` по сервисам;
* volumes: models, logs, spelling, speaker_roles;
* DNS (`127.0.0.11` + публичные);
* nvidia runtime;
* одинаковый `REDIS_URL` у api и worker.

---

# Проверь security

* `.env` не в git;
* нет hardcoded tokens;
* `google_key.json` и подобные не коммитятся;
* логи без секретов в URL.

---

# Проверь logging

* `/logs/errors.log`, crash dumps;
* structured log lines для jobs (job_id, dedup_key, status);
* исключения не проглатываются молча.

---
