---

# Контекст проекта

Проект: **Whisper STT sidecar** (v1.0.0).

GPU-sidecar: POST `/transcribe` → Redis → whisper-worker → GET `/jobs/*`.

```text
Client → whisper-stt (api) → Redis → whisper-worker (GPU) → result
```

Стек: Python, FastAPI, Redis, faster-whisper, pyannote (опц.), Docker Compose, CUDA.

**Нет:** MySQL, Alembic, React, RQ, отдельного frontend.

---

# Главная задача

Документация должна:

* соответствовать коду;
* описывать реальные сервисы compose (`redis`, `whisper-stt`, `whisper-worker`);
* отражать актуальные API и env;
* не содержать артеfactов других проектов (Channel Analytics, IPTV, MySQL, …).

---

# Критическое правило

```text
Документация должна отражать реальную систему, а не желаемую.
```

Нельзя:

* придумывать endpoints;
* описывать несуществующие сервисы;
* писать «future» как текущую реализацию;
* копировать docs из других проектов без проверки.

Source of truth = **код**.

# Обязательно сверь с кодом

* `app/main.py` — routes, auth, job status;
* `app/worker.py` — queue, recovery, GPU;
* `app/transcribe_pipeline.py` — режимы url / dual / sync;

* `app/transcribe_body.py`, `app/transcribe_schemas.py` — API contract;
* `app/settings.py`, `.env.example` — env;
* `docker-compose.yml` — services, volumes, env;

# Docker documentation

* 3 сервиса: redis, whisper-stt, whisper-worker;
* port `WHISPER_HOST_PORT` (default 19900);
* volumes: models, logs, spelling_config, speaker_roles_config;
* `WHISPER_PROCESS_ROLE` per service;
* rebuild после изменений backend.
