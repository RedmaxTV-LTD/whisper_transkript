# 07 — Поиск и исправление багов

Используй этот промт для поиска и исправления багов в Whisper STT sidecar.

---

# Промт

Найди и исправь баг в проекте Whisper STT sidecar:

```text
<ОПИСАНИЕ_БАГА>
```

---

# Контекст проекта

```text
POST /transcribe → Redis queue → whisper-worker (GPU) → GET /jobs/*
```

Критическое правило:

```text
HTTP API must never block on GPU transcription.
```

Исправление не должно:

* переносить STT в API как «быстрый fix»;
* ломать dedup и очередь;
* терять jobs при рестарте без документированного поведения;
* скрывать ошибки без логирования.

---

# Главная задача

* найти **root cause**;
* оценить impact (api, worker, redis, GPU, клиент n8n);
* минимальный безопасный fix;
* проверить побочные эффекты.

```text
Не исправляй симптомы — исправляй причину.
```

---

# Типичные области багов

## API (`main.py`)

* 503 redis unavailable;
* dedup_lock_busy;
* неверный `client_hint`;
* self-heal queue.

## Worker (`worker.py`)

* зависание на sem / waiting_gpu без heartbeat;
* orphan после restart;
* CUDA hang (pyannote + whisper load order);
* temp files не удаляются.

## Redis (`job_store.py`)

* job в `queued` но не в queue;
* duplicate LPUSH;
* stale не срабатывает;
* worker_alive TTL.

## Pipeline

* неверный sync offset;
* fallback не срабатывает;
* dual-track роли перепутаны;
* diarization без HF_TOKEN;
* download timeout / size limit.

## Audio

* ffmpeg failures;
* stereo → mono;
* corrupt WAV.

---

# Запрещённые «исправления»

* `sleep()` вместо proper retry;
* swallow exceptions;
* отключить dedup «чтобы работало»;
* sync transcribe в POST;
* удалять failed jobs без статуса.

---

# После исправления проверь

* health + worker_available;
* полный цикл job: POST → poll → completed;
* dedup при повторном POST;
* restart worker (orphan / requeue behavior);
* логи worker без новых ошибок.

```bash
docker compose logs whisper-worker --tail=100
curl -s http://localhost:19900/health
```
